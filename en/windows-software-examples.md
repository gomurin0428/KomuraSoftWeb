## What kinds of Windows software this includes

KomuraSoft handles Windows software that goes beyond simple screens alone.
That includes communication, device integration, resident processing, and coexistence with existing assets.
The detailed examples that used to be shown on the homepage are collected here.

## Main development examples

- Business desktop applications
  Windows applications for internal operations such as data entry, search, list management, and reporting.
- Equipment integration and control applications
  Applications for configuration, monitoring, and control that work with industrial cameras and peripheral devices.
- Communication clients and monitoring tools
  TCP/IP-based clients, status monitors, log collection tools, and utilities for isolating difficult field issues.
- Integration apps that reuse existing COM / ActiveX / OCX assets
  Enhancements built on existing components, new integration layers, and staged replacement strategies.
- Modernization of existing Windows software and 64-bit support
  Reorganization of x86-based software and upgrades that must account for 32-bit / 64-bit interoperability.
- Custom tools for investigation and verification
  Dedicated tools for root-cause investigation, observation, and log collection.
- Serial and USB device integration apps
  Windows applications that communicate with hardware over serial ports or USB.
- Custom protocol communication software
  Bidirectional communication programs, status monitoring, and server-side processing for real operational load.
- Windows services and resident programs
  Services and resident processes that run from OS startup for monitoring, scheduled work, and background integration.
- Multi-threaded and multi-process software
  Systems designed around parallel execution, responsibility separation, and stable long-running operation.
- Office integration tools and add-ins
  Tools and add-ins that work with Excel, Access, and PowerPoint.
- Excel-based report output systems
  Systems that generate and export Excel-based reports from business workflows.

## Common consultation themes around these systems

- reusing existing COM / ActiveX / OCX assets while still making progress
- clarifying 32-bit / 64-bit interoperability and C++/CLI boundaries
- making communication stalls and long-run failures easier to investigate
- separating UI, background work, logging, and exception design more cleanly
