## こういう Windows ソフトに対応しています

KomuraSoft では、単に画面を作るだけでなく、通信、装置連携、常駐処理、既存資産との共存まで含めた Windows ソフトを扱っています。
ホームに載せていた具体例は、このページにまとめています。

## 主な開発例

- 業務用デスクトップアプリ
  入力、検索、一覧、帳票出力など、社内業務で使う Windows アプリの設計と開発。
- 装置連携・制御アプリ
  産業用カメラや周辺機器と連携する、設定・監視・制御用アプリ。
- 通信クライアント・監視ツール
  TCP/IP 通信を使うクライアント、状態監視、ログ取得、現象切り分け用ツール。
- 既存 COM / ActiveX / OCX 資産を活かす連携アプリ
  既存コンポーネントを使った改修、新規連携部の追加、段階的な置き換え。
- 既存 Windows ソフトの改修・64bit 対応
  x86 前提の既存ソフト整理や、32bit / 64bit 相互運用を踏まえた改修。
- 調査・検証用の専用ツール
  再現しにくい不具合の原因切り分けや、ログ収集のための専用ツール。
- シリアル通信・USB連携アプリ
  シリアルポートや USB を使う設定・監視・制御アプリ。
- 独自プロトコルの通信ソフト
  双方向メッセージ連携、状態監視、高負荷を考慮した通信プログラム。
- Windows サービス・常駐プログラム
  OS 起動時から動作するサービス、監視、定期処理、バックグラウンド連携。
- マルチスレッド・マルチプロセス対応ソフト
  役割分離、並行実行、安定運用を前提とした構成。
- Office 連携ツール・アドイン
  Excel、Access、PowerPoint などと連携する業務ツールやアドイン。
- Excel 帳票出力システム
  業務システムから Excel 帳票を生成・出力する仕組み。

## 相談として多いもの

- COM / ActiveX / OCX を含む既存資産を残しながら改修したい
- 32bit / 64bit 相互運用や C++/CLI 境界を整理したい
- 通信停止や長期稼働後の障害を調査しやすい構成にしたい
- UI、バックグラウンド処理、ログ、例外設計を分けて整理したい
